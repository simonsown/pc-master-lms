1. GIT BRANCHING

   - Mỗi người tạo nhánh riêng:

     * Người kia (B): feature/2d-renderer (xử lý 2D)
   - Khi xong: tạo Pull Request → review → merge vào main

3. CÁCH DEPLOY KHÔNG XUNG ĐỘT
   - Mỗi người deploy từ nhánh của mình lên Vercel preview:
     git push origin feature/3d-viewer
     npx vercel --yes
   - Vercel tự tạo URL preview riêng (vd: pc-master-lms-git-feature-3d-viewer.vercel.app)
   - Chỉ merge vào main khi cả 2 tính năng hoàn thành
   - Deploy production: merge vào main → push → Vercel auto-deploy

4. LỆNH NHANH
   git checkout -b feature/3d-viewer    # tạo nhánh mới
   git add . && git commit -m "msg"     # commit
   git push origin feature/3d-viewer    # push lên GitHub
   npx vercel --prod                    # deploy lên Vercel preview (từ nhánh hiện tại)

5. KIỂM TRA XUNG ĐỘT TRƯỚC KHI MERGE
   git checkout main
   git pull
   git checkout feature/3d-viewer
   git merge main      # merge main vào nhánh, fix conflict nếu có
   # test kỹ trước khi tạo PR
